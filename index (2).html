How FloodGuard Works

From SOS to Safe —
in Minutes

Three roles. One platform. Every second of response time accounted for.

📱
Civilians Report

Anyone can submit a geotagged SOS from any device — no account required. The form captures location via GPS, severity, people count, and special needs like boats or medical help.

No login needed GPS capture Works on 2G
⚡
System Dispatches

The moment an SOS arrives, it's broadcast via Kafka and WebSocket to every ops dashboard. The auto-dispatch engine finds the nearest qualified volunteer — certified rescue + boat if needed — and assigns them instantly.

Real-time Kafka Auto-assign Haversine routing
🤝
Volunteers Respond

Field volunteers receive their assignment, navigate to the location, and check in live via GPS. Coordinators watch every deployment on the live map and can re-assign if situations change.

Live GPS tracking Role-based access Offline fallback
Platform Capabilities

Built for the chaos
of real disasters

🗺️ Geospatial Live Map
Leaflet-powered real-time map shows flood zones (colour-coded by severity), active SOS pins, rescue boat positions, shelter locations, and volunteer GPS — all updating live via WebSocket.
LeafletPostGISWebSocket
🚨 SOS Auto-Dispatch
Civilian reports trigger immediate Kafka events. The Haversine-based spatial query finds the nearest qualified volunteer — optionally filtering for boat operators or medical-trained responders.
KafkaPriority Queue
👥 Volunteer Coordination
Self-registration with skill profiles (rescue, medical, boat, 4WD). Zone assignment, live GPS check-ins, status tracking (Active / Deployed / Standby), and per-volunteer incident history.
Skill matchingLive GPS
📦 Resource Allocation
Track food, water, medical supplies, rescue boats, vehicles, and power across all depots. Automatic critical-low alerts fire when stock crosses thresholds — with Kafka broadcast and WebSocket dashboard push.
Threshold alertsScheduled checks
🔐 Secure Role-Based Access
JWT-authenticated ops dashboard with three roles: Admin, Coordinator, and Field Officer. Civilians submit SOS with zero friction — no account, no barrier to reaching help.
Spring SecurityJWTRBAC
📡 Real-Time Everything
STOMP WebSocket broker pushes all incident updates, volunteer movements, and resource changes instantly to every connected dashboard. No polling, no stale data during a crisis.
STOMPSockJS<100ms latency
